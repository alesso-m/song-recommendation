Spotify Music Recommender

By Ethan and Alesso

Our Initial Goal

Many people have trouble finding new music they like, and often come across songs they like by chance. So, with a music recommender, people will be able to use significantly less time to find songs that they like. For this project, we used Spotify, a popular music app that records the streaming history of each user. We used our Spotify data to put into Machine Learning algorithms to create a recommendation system. We used Machine Learning because the recommendations will become more accurate for each song listened, and people listen to a lot songs. For ease of use, we wanted the end product to be a website.

Why We Chose This

We both use Spotify and were interested in creating our own music recommender. We thought it would be a useful learning experience to code this on our own and learn how to do it with Python. We were interested in learning Python and Machine Learning and applying our code and research to create this. We think music is a popular topic that everyone enjoys, and recommending things online automatically can be difficult. For example, Netflix is very invested in their recommendation system, and they offered $1 million to anyone who could create a better recommendation system. But Netflix never implemented the winning ideas due to engineering costs, which shows that recommending things automatically isn't so simple.

Flask and Virtual Environment

In our coding we used Flask, a web framework. Flask provides you with tools, libraries and technologies that allow you to build a web application. We needed to learn and implement Flask, which is in Python, in order to combine our Python algorithm with our HTML page. What Flask enabled us to do was save the input from the form on our website as a variable in Python, getting Python variables from our HTML form. With these variables we could then run them in our Python algorithm, and then ultimately print values from the resulting Python list on our website. We used a virtual environment as well. A virtual environment is a tool that helps to keep dependencies required by different projects separate by creating isolated python virtual environments for them. This is one of the most important tools that most of the Python developers use. Virtual environments isolate the python project so that the dependencies of every project are isolated from the system and each other. For example, when I send a python file to Ethan, it will try to use the same file directories that were for my computer, and may not work well since he has them in different locations, but a virtual environment as all the installations within it, so by using the virtual environment we could collaborate and easily send working code files.
